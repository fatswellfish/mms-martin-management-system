# 项目规划: MMS 综合管理系统 (模块化架构)

## 项目目标与范围
- **总体目标**: 构建一个稳定、可扩展的农场综合管理系统，实现从生产计划到运营分析的全流程数字化。
- **核心功能**:
  - 建立完整的农场层级结构（农场 -> 猪舍 -> 猪栏）
  - 实现生产批次的全生命周期管理（入栏、喂养、出栏）
  - 提供实时的数据可视化和报表功能
  - 支持多角色权限控制和操作审计追踪

## 模块化设计原则
1. **单一职责**: 每个模块只负责一个特定的业务领域。
2. **高内聚低耦合**: 模块内部功能紧密相关，模块之间通过明确定义的接口进行通信。
3. **可复用性**: 核心服务（如 `system_core`）应被所有模块共享和复用。
4. **独立部署**: 各模块应能独立开发、测试和部署。

## 模块划分与依赖关系
| 模块 | 所属类别 | 关键依赖 | 主要功能 |
|------|----------|----------|----------|
| FieldOps | Q1: 组织与规划 | system_core | 农场/猪舍/猪栏的结构管理，数据树状展示 |
| Q2_support_processes | Q2: 支持流程 | system_core, FieldOps | 设备维护、物料采购等支持性任务管理 |
| Q3_operational_processes | Q3: 运营流程 | system_core, FieldOps, Q2_support_processes | 饲料投喂、批次跟踪、健康监测等核心生产活动 |
| Q4_performance_and_improvement | Q4: 性能与改进 | system_core, FieldOps, Q2_support_processes, Q3_operational_processes | KPI分析、报表生成、持续改进流程 |
| mms/fieldops/event_engine | 特殊模块 | system_core, FieldOps | 跨模块事件的统一处理中心 |

## 开发流程与里程碑
1. **第一阶段 (1周)**: 完成系统架构设计，搭建基础代码框架，创建所有必要的目录和初始文件。
2. **第二阶段 (2周)**: 逐个完成各模块的开发，先从 `FieldOps` 和 `event_engine` 开始，因为它们是其他模块的基础。
3. **第三阶段 (1周)**: 实现模块间的集成，编写自动化测试，并进行端到端的联调。
4. **第四阶段 (1周)**: 编写详细的文档，进行性能优化，准备最终的部署包。

## 技术栈选择
- **后端**: Python + FastAPI (高性能异步框架)
- **数据库**: PostgreSQL (关系型数据库，支持复杂查询)
- **前端**: HTML/CSS/JavaScript + Jinja2 模板引擎 (轻量级，易于集成)
- **ORM**: SQLAlchemy (成熟的对象关系映射工具)
- **部署**: Docker + Nginx (容器化部署，易于扩展)

## 交付物清单 (Final Deliverables)
- 一个功能完备、结构清晰的 `mms` 项目代码仓库。
- 一份详尽的 `docs/` 目录下的系统设计文档和 API 文档。
- 一套完整的单元测试和集成测试脚本。
- 一个 `deploy.sh` 部署脚本，能够一键启动整个系统。
