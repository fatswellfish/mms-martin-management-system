# 项目需求: MMS 综合管理系统 (模块化架构)

## 项目概述
MMS (Martin Management System) 是一个基于 ISO9001 质量管理体系的综合性组织管理系统。系统采用模块化设计，分为支持模块（Core）和四个主功能模块（Q1-Q4），以确保管理流程的标准化和可追溯性。

## 核心系统 (Core)
- **定位**: 提供系统基础功能和通用服务，是整个系统的基石。
- **包含**:
  - `models.py`: 定义所有模块共享的数据模型（如用户、权限）
  - `schemas.py`: 定义数据传输对象（DTO），用于API请求/响应格式化。
  - `repository.py`: 提供通用的数据访问层接口，支持多种数据库。
  - `service.py`: 封装业务逻辑，是各模块调用的核心服务。
  - `router.py`: 提供全局路由，管理 `/system-core` 前缀下的所有端点。

## 业务流程模块 (Q1-Q4)
### Q1: 组织与规划 (Organization & Planning)
- **目标**: 管理组织结构和资源规划，符合 ISO9001 标准第5章要求。
- **关键功能**:
  - 农场层级结构管理（农场 -> 猪舍 -> 猪栏）
  - 资源分配与调度计划
  - 人员与角色权限配置

### Q2: 支持流程 (Support Processes)
- **目标**: 处理日常运营中的支持性任务，符合 ISO9001 标准第6章要求。
- **关键功能**:
  - 设备维护与保养工单管理
  - 物料采购与库存管理
  - 人力资源与排班管理
  - 财务报销与预算控制

### Q3: 运营流程 (Operational Processes)
- **目标**: 管理核心生产活动，符合 ISO9001 标准第7章要求。
- **关键功能**:
  - 饲料配比与投喂计划执行
  - 生产批次（如育肥、产房）的全流程跟踪
  - 健康监测与疾病预警系统
  - 产品出库与物流配送管理

### Q4: 性能与改进 (Performance & Improvement)
- **目标**: 分析数据，驱动持续优化，符合 ISO9001 标准第9章要求。
- **关键功能**:
  - 关键绩效指标（KPI）仪表盘
  - 运营数据分析报告生成
  - 问题根因分析（RCA）工具
  - 持续改进（Kaizen）流程管理

## 特殊模块: 事件引擎 (Event Engine)
- **定位**: 作为 `fieldops` 模块下的子模块，负责协调养殖场内部的各类事件。
- **核心职责**:
  - 接收来自 `fieldops` 内部的事件（如"健康检查完成"、"饲料到货"）
  - 对事件进行统一的分类、记录和状态管理。
  - 触发后续的自动化流程或通知。
  - 提供事件历史查询和统计分析功能。
- **技术实现**:
  - 位于 `mms/fieldops/event_engine/`
  - 使用 FastAPI 构建独立的微服务。
  - 包含完整的前端页面（列表、详情、创建）和后端逻辑。
  - 通过 `GET /api/related-data` 接口动态加载关联的畜舍和批次数据。

## 项目交付物 (Deliverables)
1. 一个清晰的、可扩展的模块化代码仓库结构。
2. 所有模块的 API 文档（使用 Swagger UI）。
3. 一份详细的系统设计文档，说明模块间的关系和数据流。
4. 一套完整的单元测试和集成测试用例。
5. 一个部署脚本，能够一键启动整个系统。
