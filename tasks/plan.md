# MMS 项目最终规划: 综合管理系统 (模块化架构)

## 项目核心定位
MMS (Martin Management System) 是一个集组织管理、运营流程、支持服务与性能分析于一体的综合性企业级系统。它不仅仅是简单的数据记录工具，而是一个驱动业务决策和持续改进的智能平台。

## 模块化架构总览
- **核心系统 (System Core)**: 提供所有模块共享的基础能力，是整个系统的基石。
- **业务流程模块 (Q1-Q4)**: 按照业务领域划分，每个模块解决一类特定问题。
- **特殊模块 (Event Engine)**: 作为跨领域的事件中枢，连接并协调所有业务模块的活动。

## 核心模块功能精要
### fieldops (组织与规划)
- **功能**: 管理农场的物理结构（农场 -> 猪舍 -> 猪栏）和生产批次的逻辑关系。
- **关键特性**:
  - 实现完整的层级树状数据结构。
  - 提供 `GET /fieldops/tree` 接口，返回可直接用于前端渲染的嵌套对象。
  - 通过 `GET /fieldops/barn/{barn_id}/pens` 获取特定猪舍的所有猪栏信息。
  - **事件引擎 (Event Engine)**: 作为养殖场内部的“神经系统”，处理来自任何地方的事件，如健康检查、疫苗接种、喂养、清洁、转移等。

### Event Engine (事件引擎)
- **定位**: 作为 `fieldops` 模块下的子模块，负责协调养殖场内部的各类事件。
- **核心职责**:
  - 接收来自 `fieldops` 内部的事件（如"健康检查完成"、"饲料到货"）
  - 对事件进行统一的分类、记录和状态管理。
  - 触发后续的自动化流程或通知。
  - 提供事件历史查询和统计分析功能。
- **技术实现**:
  - 位于 `mms/fieldops/event_engine/`
  - 使用 FastAPI 构建独立的微服务。
  - 包含完整的前端页面（列表、详情、创建）和后端逻辑。
  - 通过 `GET /api/related-data` 接口动态加载关联的畜舍和批次数据。

## 项目实施策略
1. **先建框架，后填内容**: 首先确保所有目录结构和初始文件（如 `__init__.py`, `router.py`）都已就位，再逐步填充具体代码。
2. **从基础开始**: 优先完成 `system_core` 和 `fieldops` 模块，因为它们是其他模块的依赖基础。
3. **并行开发**: 在 `fieldops` 和 `event_engine` 完成后，可以并行开发 `Q2`, `Q3`, `Q4` 模块。
4. **持续集成**: 在开发过程中，定期进行集成测试，确保各模块能协同工作。

## 最终交付物确认
- `mms/` 目录下的完整项目结构，包含所有要求的子目录和文件。
- `docs/` 目录下包含一份清晰的系统设计文档，详细说明了模块划分、接口定义和数据流图。
- `tasks/` 目录中保留了 `task1.md`, `task2.md`, `plan.md` 三份核心文档，作为项目的历史记录和决策依据。

> **总结**: 本项目的核心思想是构建一个“以事件为驱动”的智能系统。`fieldops` 定义了系统的“骨骼”（结构），`Event Engine` 则赋予了系统“血液”和“神经”（实时动态），最终通过 `Q1-Q4` 的模块来实现完整的“器官”功能，共同构成一个有机的整体。
